# Тестовые сценарии DocGen Lite

В проект добавлены unit-тесты и отдельные тестовые сценарии. Unit-тесты проверяют основные классы и функции, а сценарии имитируют типовые действия пользователя.

## Unit-тесты

Файл `docgen_unit_tests` собирается в директорию `build/tests` и покрывает следующие модули:

1. `StringUtils`:
   - обрезка пробелов;
   - экранирование HTML;
   - получение расширения файла;
   - очистка комментариев `///` и `/** */`;
   - извлечение имени функции;
   - извлечение имени класса;
   - проверка поддерживаемых расширений.

2. `FileScanner`:
   - обработка отсутствующего пути;
   - обработка одного файла;
   - игнорирование неподдерживаемых файлов;
   - рекурсивный обход папки;
   - фильтрация по расширению;
   - сортировка результата;
   - пустая директория.

3. `FileIterator`:
   - пустая коллекция;
   - непустая коллекция;
   - сохранение порядка файлов;
   - обход через range-based for;
   - корректность `begin()` и `end()`.

4. `ParserFactory`:
   - создание `CppParser` для `.cpp`, `.cc`, `.cxx`;
   - создание `HeaderParser` для `.h`, `.hpp`;
   - отклонение неподдерживаемых расширений;
   - проверка пустого и uppercase-расширения.

5. `CppParser` и `HeaderParser`:
   - извлечение документированной функции;
   - объединение нескольких строк комментария;
   - извлечение класса;
   - извлечение структуры;
   - игнорирование недокументированных сущностей;
   - обработка пустого файла;
   - обработка отсутствующего файла;
   - многострочная сигнатура функции;
   - игнорирование управляющих конструкций;
   - несколько сущностей в одном файле.

6. `HtmlBuilder`:
   - создание HTML-скелета;
   - экранирование заголовков и текста;
   - генерация заголовков разных уровней;
   - генерация навигации;
   - генерация блоков функций и классов;
   - сообщение для пустого файла.

7. `GenerationStrategy` и `GenerationStrategyFactory`:
   - выбор простой стратегии;
   - выбор расширенной стратегии;
   - fallback на простую стратегию;
   - генерация `index.html`;
   - создание вложенной папки результата;
   - экранирование HTML;
   - обработка пустой модели.

8. `GenerationSubject`:
   - уведомления без наблюдателей;
   - игнорирование `nullptr`;
   - уведомление одного наблюдателя;
   - уведомление нескольких наблюдателей;
   - сохранение порядка сообщений.

9. `DocumentationGenerator`:
   - end-to-end простая генерация;
   - end-to-end расширенная генерация;
   - прогресс через observer;
   - ошибка при отсутствии C++ файлов;
   - ошибка при отсутствующем входном пути;
   - фильтрация по типу файла;
   - поддержка `.hpp`.

## Отдельные мини-программы сценариев

Сценарии находятся в `tests/scenarios` и собираются в отдельные исполняемые файлы в `build/tests`.

### `scenario_simple_generation`

Проверяет базовый пользовательский сценарий:

1. создаётся временный проект с одним `.cpp` файлом;
2. запускается простая генерация;
3. проверяется наличие `index.html`;
4. проверяется, что комментарий функции попал в HTML.

### `scenario_advanced_generation`

Проверяет расширенную генерацию:

1. создаётся проект с `.h` и `.cpp`;
2. запускается режим `advanced`;
3. проверяется навигация;
4. проверяется, что классы и функции попали в HTML.

### `scenario_empty_project`

Проверяет поведение при пустом проекте:

1. создаётся пустая директория;
2. запускается генератор;
3. ожидается контролируемая ошибка, а не падение программы.

### `scenario_unsupported_files`

Проверяет проект без C++ файлов:

1. создаётся директория с `.md` и `.py`;
2. запускается генератор;
3. ожидается ошибка о том, что поддерживаемые C++ файлы не найдены.

## Запуск тестов

```bash
cmake -S . -B build
cmake --build build
cd build
ctest --output-on-failure
```

Также можно запускать исполняемые файлы напрямую:

```bash
./tests/docgen_unit_tests
./tests/scenario_simple_generation
./tests/scenario_advanced_generation
./tests/scenario_empty_project
./tests/scenario_unsupported_files
```
